## Workflow-A：文本预处理（手动搭建步骤）

```
应用名称：文章转视频 - 文本预处理
```

### 步骤 1：配置开始节点

点击开始节点，添加 4 个输入变量：

| 变量名 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| input_text | 段落 | 是 | 用户粘贴的全文 |
| split_mode | 单选 | 是 | 选项：auto, manual |
| project_name | 文本 | 否 | 默认值：未命名项目 |
| style_hint | 文本 | 否 | 画风偏好 |

### 步骤 2：添加条件分支节点

从开始节点连线出来，添加一个 **IF/ELSE** 节点：
- 节点名称：判断拆分模式
- 条件：`split_mode` **等于** `auto`
- IF 分支（是）→ 连接到步骤 3A
- ELSE 分支（否）→ 连接到步骤 3B

### 步骤 3A：添加 LLM 节点（自动拆分）

- 节点名称：LLM 自动拆分
- 模型：选择你已配置的 LLM
- 系统提示词：

```
你是一个专业的文章结构分析师。你的任务是将长文按自然段落或场景切分。

切分原则：
1. 每段包含一个完整场景或情节片段
2. 每段 200-800 字，太长继续拆
3. 在场景切换、时间跳跃、地点变化处切分
4. 保持对话完整性
5. 如有明确章节标记，优先按章节切分
```

- 用户提示词：

```
请将以下文章切分为段落，严格按 JSON 格式输出，不要输出任何其他内容：

{"segments": [{"index": 1, "title": "段落标题", "content": "段落正文", "summary": "一句话概括"}]}

文章内容：
{{input_text}}
```

- 输出变量名：`auto_split_result`

### 步骤 3B：添加代码节点（手动切分）

- 节点名称：手动标记切分
- 输入变量：`input_text` 引用开始节点的 `input_text`
- Python 代码：

```python
import re
import json

def main(input_text: str) -> dict:
    pattern = r'===\[([^\]]*)\]===|===SPLIT==='
    parts = re.split(pattern, input_text)
    segments = []
    index = 1
    i = 0
    current_title = ""

    while i < len(parts):
        if parts[i] is None:
            i += 1
            continue
        if i % 2 == 1:
            current_title = parts[i] if parts[i] else ""
            i += 1
            continue
        content = parts[i].strip()
        if content:
            title = current_title if current_title else f"段落 {index}"
            segments.append({
                "index": index,
                "title": title,
                "content": content,
                "summary": content[:50] + "..." if len(content) > 50 else content,
            })
            index += 1
            current_title = ""
        i += 1

    return {
        "segments_json": json.dumps(segments, ensure_ascii=False),
        "parse_success": "true" if segments else "false",
        "segment_count": str(len(segments)),
    }
```

- 输出变量：`segments_json`, `parse_success`, `segment_count`

### 步骤 4A：添加代码节点（解析自动拆分结果）

- 节点名称：解析自动拆分
- 输入变量：`auto_split_result` 引用 LLM 节点的输出
- Python 代码：

```python
import json
import re

def main(auto_split_result: str) -> dict:
    try:
        json_match = re.search(r'\{[\s\S]*\}', auto_split_result)
        if json_match:
            data = json.loads(json_match.group())
        else:
            data = json.loads(auto_split_result)

        segments = data.get("segments", [])
        if not segments:
            return {"segments_json": "[]", "parse_success": "false", "segment_count": "0"}

        for seg in segments:
            seg.setdefault("index", 0)
            seg.setdefault("title", "未命名段落")
            seg.setdefault("content", "")
            seg.setdefault("summary", "")

        return {
            "segments_json": json.dumps(segments, ensure_ascii=False),
            "parse_success": "true",
            "segment_count": str(len(segments)),
        }
    except Exception:
        return {"segments_json": "[]", "parse_success": "false", "segment_count": "0"}
```

- 输出变量：`segments_json`, `parse_success`, `segment_count`

### 步骤 5：添加条件分支（解析是否成功）

- 节点名称：解析成功？
- 条件：`parse_success` **等于** `true`
- IF → 连接到步骤 6
- ELSE → 连接到结束节点，输出"文本解析失败"

### 步骤 6：添加 HTTP 请求节点（创建项目）

- 节点名称：创建项目
- 方法：POST
- URL：`http://localhost:5005/api/projects`
- Headers：`Content-Type: application/json`
- Body 类型选 JSON，内容：

```json
{
  "name": "{{project_name}}",
  "source_text": "{{input_text}}",
  "split_mode": "{{split_mode}}",
  "style_hint": "{{style_hint}}"
}
```

- 输出变量：`project_response`

### 步骤 7：添加代码节点（提取项目 ID）

- 节点名称：提取项目ID
- 输入：`project_response` 引用 HTTP 节点输出
- 代码：

```python
import json

def main(project_response: str) -> dict:
    try:
        data = json.loads(project_response)
        return {"project_id": data.get("id", "")}
    except Exception:
        return {"project_id": ""}
```

- 输出：`project_id`

### 步骤 8：添加代码节点（准备段落列表为数组）

Dify 的迭代节点需要数组输入，先转换：

- 节点名称：准备段落列表
- 输入：`segments_json`
- 代码：

```python

    segments = json.loads(segments_json)
    return {"segment_list": segments}
```

- 输出：`segment_list`

### 步骤 9：添加迭代节点

- 节点名称：循环遍历段落
- 输入数组：引用 `segment_list`

### 步骤 10：迭代内部添加 3 个节点

**节点 10.1：LLM 生成分镜**

- 系统提示词：

```
你是一个专业的分镜脚本编剧。你将把一段文字转化为可视化的分镜脚本。
```

- 用户提示词：

```
请将以下段落转化为分镜脚本。每个分镜代表一个画面。一段文字通常拆分为 3-8 个分镜。

段落标题：{{item.title}}
段落内容：{{item.content}}

严格按 JSON 格式输出，不要输出任何其他内容：
{"shots": [{"shot_index": 1, "scene_description": "画面场景的中文描述", "prompt": "English prompt for image generation, detailed, including character appearance, scene, lighting, mood", "negative_prompt": "blurry, low quality, distorted, deformed", "dialogue": "角色台词", "narrator_text": "旁白文字", "speaker": "说话人姓名", "camera": "近景/中景/远景/特写", "characters": ["出场人物"], "duration_hint": 5}]}
```

**节点 10.2：代码解析分镜**

- 输入：LLM 的输出
- 代码：

```python
import json
import re

def main(shot_result: str) -> dict:
    try:
        json_match = re.search(r'\{[\s\S]*\}', shot_result)
        if json_match:
            data = json.loads(json_match.group())
        else:
            data = json.loads(shot_result)
        shots = data.get("shots", [])
        for shot in shots:
            shot.setdefault("shot_index", 0)
            shot.setdefault("scene_description", "")
            shot.setdefault("prompt", "")
            shot.setdefault("negative_prompt", "blurry, low quality, distorted, deformed")
            shot.setdefault("dialogue", "")
            shot.setdefault("narrator_text", "")
            shot.setdefault("speaker", "")
            shot.setdefault("camera", "")
            shot.setdefault("characters", [])
            shot.setdefault("duration_hint", 5)
        return {"shots_json": json.dumps(shots, ensure_ascii=False), "shot_count": str(len(shots))}
    except Exception:
        return {"shots_json": "[]", "shot_count": "0"}
```

**节点 10.3：HTTP 请求保存段落分镜**

- 方法：POST
- URL：`http://localhost:5005/api/projects/{{project_id}}/segments`
- Body（JSON）：

```json
{
  "segment_index": {{item.index}},
  "title": "{{item.title}}",
  "content": "{{item.content}}",
  "summary": "{{item.summary}}",
  "shots": {{shots_json}}
}
```

### 步骤 11：迭代结束后添加结束节点

- 输出：文本预处理完成，项目 ID、段落数量

**5.1 代码构造提示词**

```python
def main(item: dict, style_hint: str = "") -> dict:
    prompt = item.get("prompt", "")
    if style_hint:
        prompt = f"{prompt}, {style_hint}"
    return {
        "final_prompt": prompt,
        "final_negative": item.get("negative_prompt", "blurry, low quality"),
        "shot_id": item.get("id", ""),
        "has_text": "true" if (item.get("dialogue") or item.get("narrator_text")) else "false",

        "tts_speaker": item.get("speaker", ""),

```

> 注意：Dify 迭代内部可能无法直接引用外部变量（如 style_hint）。如果不行，把 style_hint 作为输入变量传入迭代节点，或者在开始节点直接设一个固定值。

import json

def main(image_response: str) -> dict:
    try:
        data = json.loads(image_response)
        # 适配不同引擎的返回格式，按需修改
        image_url = data.get("image_url", "")
        if not image_url:
            # ComfyUI 格式适配
            output = data.get("output", {})
            if isinstance(output, dict):
                images = output.get("images", [])
                if images:
                    image_url = images[0]

### 步骤 4：代码构造合成请求

```python
import json

def main(confirmed_response: str, segment_id: str, output_format: str = "horizontal_16_9", title_text: str = "") -> dict:
    data = json.loads(confirmed_response)
    shots = data.get("shots", [])
    compose_shots = []
    for shot in shots:
        compose_shots.append({
            "shot_id": shot.get("id", ""),
            "shot_index": shot.get("shot_index", 0),
            "image_path": shot.get("image_path", ""),
            "audio_path": shot.get("audio_path", ""),
            "audio_duration": shot.get("audio_duration", 5.0),
            "dialogue": shot.get("dialogue", ""),
            "narrator_text": shot.get("narrator_text", ""),
            "speaker": shot.get("speaker", ""),
        })
    title_config = {"enabled": bool(title_text), "text": title_text, "duration": 3.0}
    request_body = {
        "segment_id": segment_id,
        "shots": compose_shots,
        "output_format": output_format,
        "subtitle": {"enabled": True, "font_size": 24},
        "transition": {"type": "fade", "duration": 0.5},
        "title": title_config,
    }
    return {"compose_body": json.dumps(request_body, ensure_ascii=False)}
```

### 步骤 5：HTTP 调用视频合成

## 快速模式（手动搭建步骤）

```
应用名称：文章转视频 - 快速模式
```

快速模式的节点最多，但每个节点的逻辑在上面的工作流中都出现过。核心串联顺序：

```
开始（input_text, style_hint）
  → 条件：文本 < 200 字？
    → 是：LLM 扩写 → 代码选择最终文本
    → 否：直接用原文
  → LLM 生成分镜脚本
  → 代码解析分镜 + 调用项目管理 API 创建项目并保存
  → HTTP 获取分镜列表
  → 迭代：对每个分镜
      → 构造提示词 → 生图 → 保存图片 → TTS → 保存音频
  → HTTP 自动确认所有分镜
  → 代码构造合成请求
  → HTTP 调用合成
  → 代码提取结果
  → HTTP 更新段落状态
  → 结束：输出视频路径
```

每个节点的代码和配置与上面的工作流中的对应节点完全一致，不再重复。

## 总结

```
正确的搭建方式：
  1. 在 Dify 中创建"工作流编排"应用
  2. 在可视化编辑器中按上述步骤拖拽添加节点
  3. 每个节点的配置和代码直接复制粘贴
  4. 连线、保存、测试

JSON 导入不可靠的原因：
  · Dify 的 DSL 格式是 YAML 不是 JSON
  · 格式规范随版本变化
  · 节点 ID、坐标、连线关系都有严格约束
  · 手动搭建虽然慢，但 100% 可靠
```
